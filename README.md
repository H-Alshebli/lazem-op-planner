# نظام إدارة الخطة التشغيلية — لازم

React + Firebase (Firestore) + Vercel. بدون تسجيل دخول حالياً — أي شخص عنده رابط الموقع يقدر يدخل ويعدّل نفس الخطة، والتحديثات تنعكس لحظياً لكل من يفتح الرابط.

## 1) إنشاء مشروع Firebase

1. روح إلى https://console.firebase.google.com وأنشئ مشروع جديد (مثلاً `lazem-op-plan`).
2. من القائمة الجانبية: **Build > Firestore Database > Create database**. اختر أي موقع قريب (مثل `eur3` أو `me-central1` إذا متاح)، وابدأ في **وضع الإنتاج (Production mode)**.
3. بعد الإنشاء، افتح تبويب **Rules** في Firestore والصق محتوى ملف `firestore.rules` الموجود في هذا المشروع، ثم اضغط **Publish**.
   - هذه القواعد تفتح القراءة/الكتابة على مستند الخطة فقط (بدون أي مصادقة)، وتمنع أي شيء آخر. مناسبة للاستخدام الداخلي السريع، لكنها تعني إن أي شخص يعرف رابط الموقع يقدر يعدّل البيانات — إذا احتجت لاحقاً حماية أقوى (تسجيل دخول)، قول لي وأضيفها.
4. من **Project settings (⚙️) > عام > تطبيقاتك**، أضف **تطبيق ويب (Web App)** جديد، وسجّل بيانات الإعداد (`firebaseConfig`) اللي بتظهر لك.

## 2) إعداد المشروع محلياً

```bash
npm install
cp .env.example .env
```

افتح ملف `.env` وعبّي القيم من `firebaseConfig` اللي حصلت عليها في الخطوة السابقة:

```
VITE_FIREBASE_API_KEY=...
VITE_FIREBASE_AUTH_DOMAIN=...
VITE_FIREBASE_PROJECT_ID=...
VITE_FIREBASE_STORAGE_BUCKET=...
VITE_FIREBASE_MESSAGING_SENDER_ID=...
VITE_FIREBASE_APP_ID=...
```

ثم شغّل المشروع محلياً للتجربة:

```bash
npm run dev
```

## 3) النشر على Vercel

### الطريقة الأسهل (عبر GitHub)
1. ارفع هذا المجلد كمستودع على GitHub.
2. في https://vercel.com اضغط **Add New > Project** واختر المستودع.
3. Vercel بيكتشف إنه مشروع Vite تلقائياً (Build command: `vite build`, Output: `dist`).
4. في خطوة **Environment Variables**، أضف نفس المتغيرات الستة من ملف `.env` (نفس الأسماء بالضبط).
5. اضغط **Deploy**. بعد ما يخلص، تحصل على رابط جاهز تقدر تشاركه مع فريقك.

### أو عبر Vercel CLI
```bash
npm i -g vercel
vercel
# اتبع الخطوات، وأضف متغيرات البيئة عند الطلب أو من لوحة تحكم Vercel بعدين
vercel --prod
```

## كيف يخزن النظام البيانات؟

كل بيانات الخطة (المعلومات، الرؤية، SWOT، الأهداف والإجراءات) تُخزَّن كوثيقة واحدة في Firestore على المسار:

```
lazemOperationalPlan/main
```

أي تعديل من أي شخص يُحفظ تلقائياً (بدون زر حفظ) بعد نصف ثانية من التوقف عن الكتابة، وينعكس لحظياً على كل من يفتح الصفحة في نفس الوقت (real-time sync عبر `onSnapshot`).

## تطوير لاحق مقترح

- إضافة تسجيل دخول (Firebase Auth) مع صلاحيات (مسؤول الخطة يعدّل، الباقي يشاهدون) إذا احتجت حماية أقوى.
- تصدير تقرير PDF من لوحة المتابعة.
- ربط تذكيرات تلقائية للإجراءات المتأخرة.
